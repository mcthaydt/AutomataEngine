# First Light

A minimal registered AutomataEngine game: steer the keeper (WASD/arrows) to the
beacon before the time limit. Generated by `npm run new-game` as a working
starting point — every layer below is meant to be replaced with your game.

## Architecture

- `src/sim/sim.ts` — pure fixed-step simulation. No DOM, clocks, or RNG; the
  same inputs always produce the same run, which is what makes headless
  evaluation and agent-driven tuning possible.
- `src/game/gameplay.ts` — wires the sim to the engine render port; shared by
  the browser boot and the editor's live preview.
- `src/project/` — the authoring surface: schemas, template, validation,
  compilation, evaluation, and the registry loader entries
  (`loadEditorRegistration`, `loadHeadlessRegistration`). Because these
  exist, this game appears in the level-editor chooser and the MCP server
  without any wiring elsewhere.
- `src/main.ts` — thin browser composition root (excluded from coverage).

## Commands

From the repository root:

```bash
npm run dev -w first-light            # play at http://127.0.0.1:5178
npx vitest run --project first-light  # unit tests
npm run typecheck -w first-light
npm run build -w first-light          # validates the project, then builds
PLAYWRIGHT_ONLY=first-light npx playwright test games/first-light/e2e
```

## Authoring content

The authored project lives in `public/project` (manifest, scenes, resources).
Three equivalent ways to change it:

1. Edit `src/project/template.ts`, then `npm run generate:project -w first-light`
   to regenerate `public/project`. The content test keeps both in sync.
2. Open it in the editor: `npm run dev:editor`, choose First Light.
3. Let an agent edit it over MCP:
   `node_modules/.bin/automata-editor-mcp --workspace .` then call the openProject tool with gameId "first-light"
   exposes typed add/set/validate/evaluate tools.

`npm run build -w first-light` refuses to ship an invalid project.
